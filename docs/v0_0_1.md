## 1. 项目定位

本项目是一个**教学演示级 K-Means 聚类算法实现**，面向**超大规模数据集（9+ GiB）**，
采用**多后端并行架构**（CUDA / OpenMP / Sequential），
支持**自动 K 推导**和**断点继续**。

核心特征：
- 不是生产级系统，而是用于**教学展示**并行计算概念的教学项目
- 所有配置通过 `config.json` 管理，零 CLI 参数
- 设计上性能优先于可读性，但优化思路已经记录下来放在`docs/algo/issue{int}.md`中
- 算法正确性、后端对比、并行策略演示

## 2. 核心功能拆解

### 2.1 算法层

| 功能 | 说明 |
|------|------|
| K-Means Lloyd 迭代 | 标准分配-更新循环，支持多后端 |
| 自动 K 推导 | 均匀采样 -> 多 K 搜索 → Kneedle 肘部检测 |
| 断点继续 | 二进制 checkpoint，可从中断处恢复 |

### 2.2 并行后端

| 后端 | 文件 | 定位 |
|------|------|------|
| Sequential | `kmeans_sequential.cpp` | 验证正确性，单线程参考实现 |
| OpenMP | `kmeans_omp.cpp` | CPU 多核加速，Fork-Join 模型 |
| CUDA | `kmeans_kernels.cu` | GPU 加速，流式分块处理超大文件 |

### 2.3 数据层

| 功能 | 文件 | 说明 |
|------|------|------|
| 数据集加载 | `dataset.cpp` | mmap / SoA 分离 / 流式三种策略 |
| 数据格式 | — | 纯二进制 `double[2]` 交错，小端序，每点 16 字节 |

### 2.4 配置层

| 功能 | 说明 |
|------|------|
| JSON 解析 | 手写递归下降解析器，零外部依赖 |
| 配置映射 | JSON 字段 -> KMeansConfig 结构体 |
|保留接口|每个常量都得留一个`description`的字段，用来给维护者看|


### 2.5 输出层

| 文件 | 说明 |
|------|------|
| `result.txt` | 运行日志，含配置参数和结果 |
| `labels.bin` | 每个点的聚类标签（二进制） |
| `centers.csv` | 各簇中心坐标 |
| `render.csv` | 采样点 + 标签（可视化用） |

## 3. 当前代码问题清单

### 3.1 `main.cpp` 职责过重

`src/main.cpp` 约 425 行，糅合了以下独立职责：


checkpoint.cpp 实现`中断续接`的逻辑，`config.json`必须有字段决定多少轮迭代。

main.cpp就2个核心目标，解析json文件，把配置丢给`kmeans`目录下的对应的代码跑kmeans分类。

checkpoint.cpp 实现`中断续接`的逻辑（`config.json`必须有字段决定多少轮迭代），kmeans下的代码调用，非main.cpp调用

其余小功能，比如获取时间戳，拼接文件路径，文件的输入输出……自己看着办。

## 4. 设计分析


### 4.2 设计原则对照

**单一职责原则**：违反。`main.cpp` 承担了路径、配置、输出、编排四个职责。

**开闭原则**：基本符合。新增后端只需添加新的 `run_*` 函数，不改动现有代码。

**依赖倒置**：部分符合。`KMeans` 依赖 `Dataset` 接口而非具体加载逻辑。但 `main.cpp` 直接依赖具体实现。

## 5. 重构方向建议

### 5.1 优先级排序

1. **路径工具独立** — 最简单的胜利，纯函数，不影响任何逻辑
2. **json_config.hpp 拆 .cpp** — 减少编译开销，不改变接口
3. **输出逻辑独立** — 清除 main.cpp 中最重的负担
4. **main.cpp 精简** — 上述三步完成后自然达成
5. **代码规范修复** — 括号、路径类型、初始化列表等

### 5.2 重构原则

- **改结构不改行为**：重构期间不改变任何运行时逻辑
- **小步提交**：每完成一个独立模块的提取就提交一次
- **保持向后兼容**：所有现有 include 路径不变

### 5.3 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| CUDA 编译环境不可用 | 高 | 先只重构 C++ 代码，确认无误后再处理 CUDA 相关 |
| 路径行为改变 | 中 | 保持原有路径解析逻辑不变，只移动位置 |
| 输出格式变化 | 低 | 输出函数复制到新文件，逐行比对确保一致 |
